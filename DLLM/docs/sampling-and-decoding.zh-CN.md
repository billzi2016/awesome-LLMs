# Diffusion LLM 的采样与解码

本文档聚焦 Diffusion LLM 在推理阶段如何生成文本。核心点很简单：

\[
\text{training objective} \neq \text{decoding strategy}.
\]

即便两个 DLLM 的训练损失很接近，它们在推理阶段的行为也可能完全不同，关键差异来自：

- 去噪步数用了多少；
- 是同步解码还是异步解码；
- 会不会把低置信度 token 重新 mask；
- 是全并行生成，还是分块生成；
- 是否在 refinement 过程中显式注入左到右偏置。

## 1. 把解码看成反向腐蚀

记 \(x_T\) 为完全腐蚀状态，\(x_0\) 为最终生成文本。DLLM 的解码过程就是应用一个学到的反向过程：

\[
x_T \rightarrow x_{T-1} \rightarrow \cdots \rightarrow x_0.
\]

在图像 diffusion 中，这通常意味着反转高斯噪声；在语言里，尤其是 masked diffusion 中，这通常意味着逐步把 `[MASK]` 替换成真实 token。

一般性的解码问题可以写成：

\[
\text{实际中应该怎样选择 } p_\theta(x_{t-1}\mid x_t) \text{？}
\]

这个问题在工程上会进一步拆成：

- 每一步是不是都要预测所有 token 位置？
- 某个 token 一旦预测出来，是否永久锁定？
- 低置信度 token 是否允许后续再修订？
- 所有 token 是否以同样速度穿过 diffusion time？

## 2. 同步并行解码

最简单的 masked DLLM 解码器是完全同步的。

### 2.1 基本过程

从全 mask 序列开始：

\[
x_T = [m, m, \dots, m].
\]

在每个步骤 \(t\)，对所有 masked 位置预测 token 分布：

\[
\pi_\theta(\cdot \mid x_t, t).
\]

然后有几种做法：

- 对所有 masked 位置直接贪心填充；
- 从分布中采样；
- 只填一部分位置，然后继续细化。

### 2.2 为什么它能并行

和自回归模型不同，这里的预测是条件在整个“部分去噪序列”上的：

\[
p_\theta(x_0 \mid x_t, t),
\]

因此所有位置都可以通过双向注意力同时更新。

优点是长文本时墙钟延迟更低；缺点是早期 token 决策可能是在上下文还不充分时做出的。

## 3. 重掩码与迭代细化

现代 DLLM 解码的一个核心思想，是 token 预测结果应该允许被修订。

### 3.1 基于置信度的重掩码

记位置 \(i\) 的置信度为

\[
c_i = \max_{v \in \mathcal{V}} \pi_\theta(v \mid x_t, t).
\]

一个常见的重掩码策略是：

1. 对候选位置预测 token；
2. 保留高置信度位置；
3. 把低置信度位置重新 mask；
4. 继续去噪。

这样解码就不再是一次性填空，而是自适应细化。

### 3.2 为什么重掩码有效

在解码早期，上下文并不完整。某个 token 看上去局部很自信，但可能在全局上并不一致。重掩码给了模型后续修订的机会。

这在以下场景尤其重要：

- 代码生成；
- 强推理任务；
- 长文本输出；
- 对全局一致性要求高的任务。

### 3.3 一个有用的抽象

定义位置 \(i\) 在步骤 \(t\) 的保留决策 \(k_i^{(t)} \in \{0,1\}\)：

\[
k_i^{(t)} =
\begin{cases}
1, & c_i \ge \tau_t \\
0, & c_i < \tau_t
\end{cases}
\]

其中 \( \tau_t \) 是与步骤有关的阈值。于是解码就变成在两件事之间交替：

- token 预测；
- 保留 / 重新 mask 决策。

这是理解 masked diffusion decoding 最干净的一种方式。

## 4. 固定日程与自适应日程

并不是所有去噪日程都一样。

### 4.1 固定日程

最简单的情况下，未解开的 mask 数量按预先定义好的日程递减：

\[
M_{t-1} = \rho_t M_t,
\]

其中 \(M_t\) 表示步骤 \(t\) 时还处于 mask 状态的位置数，且 \(0 < \rho_t < 1\)。

这种方法简单高效，但忽略了 token 难度差异。

### 4.2 自适应日程

自适应日程让模型自己决定哪些位置还需要继续演化。也就是说，除了 timestep \(t\) 之外，解码器还会使用置信度或不确定性信号。

概念上可以写成：

\[
\text{位置 } i \text{ 的推进速度}
=
f(t, c_i, \text{context sensitivity}, \dots).
\]

这种做法能把计算预算分配得更合理：

- 容易的位置更早结束；
- 困难的位置获得更多细化步数。

## 5. 半自回归与分块解码

Diffusion decoding 不一定非得全并行。

### 5.1 分块生成

有些方法会把序列切成多个 block，让一部分 block 更早完成去噪，另一部分 block 更晚完成。这样就落在两种极端之间：

- 完全自回归解码；
- 完全并行 masked 解码。

若 block 编号为 \(b = 1, \dots, B\)，可以用一个简化写法表示：

\[
p(x) \approx \prod_{b=1}^{B} p_\theta(x^{(b)} \mid x^{(<b)}, \text{noisy state}).
\]

这并不是标准 AR 分解，因为每个 block 内部依然可能通过 diffusion 过程去噪。

### 5.2 为什么重要

分块解码有机会同时改善：

- 长程一致性；
- 可控性；
- 速度与质量之间的权衡。

这也是为什么像 `SSD-LM` 这样的工作值得从纯 masked diffusion 中单独区分出来。

## 6. AR-Diffusion 与位置依赖的去噪

`AR-Diffusion` 很重要，因为它显式地把左到右不对称重新引了回来。

### 6.1 核心思想

不同位置分配不同的去噪预算。左边 token 更早“浮现”出来，右边 token 则可以在后续步骤里条件在左边 token 上再生成。

简化地说，位置 \(i\) 可以拥有自己的有效去噪长度：

\[
T_i \le T_j \quad \text{for } i < j.
\]

于是左侧位置会比右侧位置更早变干净。

### 6.2 如何理解

这并不意味着完全回到自回归 decoding，而是在 diffusion 过程中加入了因果偏置：

\[
\text{diffusion refinement}
+ \text{position-dependent generation order}.
\]

这样既保留了一部分 diffusion 的灵活性，也比完全同步解码更尊重语言的顺序结构。

## 7. 采样预算与质量

DLLM 的解码质量通常强依赖 refinement 步数。

### 7.1 计算量与质量的权衡

记 \(K\) 为解码迭代次数，则大体上有：

\[
K \uparrow \Rightarrow \text{更高计算成本，通常也带来更好质量}.
\]

但回报并不是线性的。超过某个点之后，多加几步可能只带来很小提升。

### 7.2 低预算下的常见失败模式

如果步数太少：

- 全局结构可能不稳定；
- 早期 token 决策可能错误；
- 输出可能流于浅层或模板化。

这也是为什么近期很多工作并不是只改训练，而是专门研究如何在相同预训练模型下、有限推理预算内拿到更高质量。

## 8. 近期解码方向

有两条近期方向尤其值得关注。

### 8.1 连续化或 Flow 风格解码

像 **Masked Diffusion Decoding as x-Prediction Flow**（2026 年 6 月 27 日）这样的新工作认为，标准二元 mask 决策太刚性了。一个位置不是“完全提交”就是“完全 masked”，中间没有表达部分置信度的空间。它把 clean-token prediction 解释为 embedding 空间中的连续细化信号，从而导向更平滑、按 token 异步推进的解码方式。[[论文](https://arxiv.org/abs/2606.29066)]

### 8.2 更鲁棒的重掩码

像 **CoRe: Context-Robust Remasking for Diffusion Language Models**（2026 年 2 月 4 日）这样的工作则质疑：仅靠 token 自己的置信度是否足够决定是否保留。它更关注某个 token 在周围上下文被轻微扰动时是否依然稳定，并优先重掩码那些“上下文脆弱”的位置。[[论文](https://arxiv.org/abs/2602.04096)]

这些方向很重要，因为它们说明 DLLM 的进展越来越多地发生在 inference-time，而不只是训练目标层面。

## 9. 与 AR 解码的实践对比

自回归解码通常：

- 更容易分析；
- 在标准 beam / sampling 配方下更稳定；
- 和 next-token 训练天然一致。

DLLM 解码通常：

- 更并行；
- 更容易支持修订；
- 更依赖日程设计；
- 更依赖置信度估计和 remasking 策略。

所以在实践上可以概括成：

\[
\text{AR decoding 优化 token 顺序},
\qquad
\text{DLLM decoding 优化 refinement 顺序}.
\]

## 10. 读 DLLM 论文时该看什么

读一篇新的 DLLM 论文时，建议先问：

1. 解码是全并行、半自回归，还是位置异步的？
2. token 一旦预测出来是否永久锁定，还是允许修订？
3. 去噪日程是固定的还是自适应的？
4. 是否单独设计了置信度或不确定性估计器？
5. 想达到较好质量大概要多少 decoding steps？
6. 提升主要来自训练、推理，还是两者都有？

这些问题通常比“模型名字是什么”更能反映一个方法的实际可用性。
