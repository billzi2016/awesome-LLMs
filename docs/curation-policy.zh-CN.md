# 收录规则

本文档定义本项目的资源收录方式。

## 收录标准

资源满足以下多数条件时，可以考虑收录：

- 与仓库声明的主题范围直接相关；
- 如果主题偏向 skill / workflow，还应体现可复用的 skill 概念、结构或实现模式；
- 有稳定且可访问的来源；
- 提供论文、代码、文档、benchmark、数据集、registry 条目、模板或技术说明；
- 对研究、工程、评估或学习有帮助；
- 如果主题偏向工作流，还应对编写、打包、运行、发现或评估该类工作流有帮助；
- 描述可以从来源中客观核验；
- 不引入版权、许可证、隐私或平台规则风险。

## 优先资源

优先收录：

- 基础论文；
- 官方实现；
- 官方文档、规范或平台内置体系；
- 持续维护的库；
- 可复现实验；
- 当其属于主题核心时，优先收录清晰的模板、manifest、打包约定或 registry；
- 常用教程或笔记；
- benchmark 和评估资源；
- 明确属于仓库主题范围的应用。

## 拒绝标准

不建议收录：

- 与仓库范围关系较弱的资源；
- 对 skill 主题来说，没有可复用结构的通用 prompt 集合；
- 对 skill 主题来说，与技能机制无明显关系的泛 AI 工具列表；
- 无法访问或无法核验的资源；
- 重复条目；
- 主要用于宣传的页面；
- 缺少技术细节的页面；
- 复制 PDF、镜像数据集、非官方模型权重或受保护的 marketplace 内容；
- 可能违反版权、许可证、隐私或平台规则的资源。

## 链接规则

优先使用原始来源，顺序如下：

1. arXiv、DOI、出版社页面、官方论文页或平台 / 提供方官方文档
2. 作者、机构或官方代码仓库
3. 官方文档、registry、marketplace 或 catalog 页面
4. 数据集、benchmark 或打包 / runtime 接入官方页面
5. 持续维护的项目页面
6. 二次整理页面

如果链接无法核验，不要加入。

## 描述风格

描述应做到：

- 事实准确；
- 简洁；
- 中性；
- 在适用时具体说明方法、任务、贡献、runtime 边界或编写价值。

避免使用“最强”“必看”“革命性”“state-of-the-art”“agent 的未来”等缺少上下文支撑的表达，除非该表述对应来源中的具体依据。

## 范围补充说明

这个总仓库同时包含偏研究型主题和偏工作流型主题。

通常适合收录的包括：

- 与主题直接相关的论文、benchmark、数据集和官方实现；
- 官方文档、规范或平台原生机制；
- 当它们本身就是主题核心时，模板、registry 和运维指南；
- 能清楚展示目标能力或工作流的具体示例。

通常不适合收录的包括：

- 与主题没有明确关系的泛背景材料；
- 看不出可复用结构的模糊资源列表；
- 无法核验的镜像或转贴的受保护内容。

## 维护

应定期检查：

- 失效链接；
- 重复条目；
- 过时描述；
- 已归档仓库；
- 已迁移论文或项目页面；
- 中英文内容不一致。
