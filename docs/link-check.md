# Link Check

Use this checklist before adding or updating resources.

## Required Checks

- The link opens successfully.
- The link points to the intended resource.
- Title, authors, venue, year, repository identity, or skill name match the linked page.
- Code or repository links point to the official author, organization, or platform page when possible.
- Dataset, model, documentation, and registry links point to canonical pages rather than mirrors or reposts.
- The resource does not require copying protected content into this repository.

## Preferred Sources

1. arXiv, DOI, publisher, official paper page, or official platform/provider documentation
2. official GitHub, organization, or platform repository
3. official project, registry, marketplace, or catalog page
4. official documentation
5. official dataset, benchmark, or packaging/runtime page

## Risk Signals

- Download-only pages with unclear ownership
- unofficial mirrors
- model weights without license information
- reposted documentation pages
- copied PDFs or course material
- private marketplace content shared indirectly
- broken redirects
- pages unrelated to the resource title

When uncertain, do not include the resource until the source is verified.
